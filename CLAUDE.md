# Указания для Claude

## Формат ответов
- Каждый ответ в чате начинай строкой с текущим московским временем: `🕐 ЧЧ:ММ (МСК)`.
  Время брать командой `TZ=Europe/Moscow date "+%H:%M"`.

## Рабочий процесс
- В начале задачи: `git pull`.
- Изменения вносятся в `index.html` / `sw.js` в корне репозитория.
- Автотесты: временный `test.js` в корне (Node), прогнать до зелёного, затем удалить.
- Каждую заметную задачу фиксировать записью в `DEVLOG.md` (обратная хронология, вверху файла).
- Пуш в `main` с описанием коммита на русском; GitHub Pages деплоится автоматически.
- НЕ пушить два раза подряд с интервалом в секунды (например, «фикс» и следом «DEVLOG-хэш»):
  каждый пуш запускает деплой Pages, первый отменяется вторым, а идущие встык деплои
  у GitHub нестабильны (зависают в Queued / падают). Делать оба коммита локально и
  отправлять ОДНИМ `git push`. Хэш коммита в DEVLOG писать во втором локальном коммите
  до пуша, либо ссылаться на заголовок коммита.
- После пуша проверять статус деплоя (GitHub Actions, workflow «pages build and
  deployment»): бывает транзиентный сбой — тогда повторить пустым коммитом.
